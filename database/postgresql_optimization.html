parallel gather , gather merge

parallel gather , gather merge 是一段并行任务中的ROOT部分，gather用来收集所有计算子进程的计算结果，gather merge用来收集所有计算子进程的计算结果并进行归并排序。

Sort

order by time
order by time desc, id desc;

Limit

limit 1000;

Append

当一个表有多个分区，查询所有分区，并按某列排序输出，常规的做法是所有的记录append，然后sort
PostgreSQL 10.0 将支持append node的并行计算，也就是说所有的分区表可以并行的sort，然后返回，此时就可以使用merge sort来提高排序的速度

template0 和template1 数据库

postgreSQL 默认会附带两个模板数据库

创建数据库如果不指定TEMPLATE，默认为template1

手动指定

diff

任何时候都不要对template0进行修改，作为一个原始干净的模板

template1 可以连接并创建对象，template0 不可以连接

使用 template1 模板库建库时不可指定新的 encoding 和 locale，而 template0 可以

template0 库和 template1 都不可删除
